5.6 Multiprotocol BLE Provisioning MCC Component

Overview

The BLE provisioning component facilitates the users in selecting a combo application. Additionally, the user can select the device role and enable sleep options, if any, based on the selected combo application option. The MCC component has certain dependencies, and the table below provides details on these dependencies and their requirements.
Table 5-1. BLE provisioning component MCC Dependencies
Dependency Component Dependency Type Description
Transparent Profile Mandatory BLE provisioning uses transparent profile as per the functional design
PDS_SubSystem Mandatory BLE provisioning uses PDS for Data backup.
IEEE 802.15.4 PHY Mandatory BLE provisioning utilizes the 15.4 PHY when the selected combination application is based on the 15.4 PHY and IEEE 802.15.4 radio services.
IEEE 802.15.4 MAC Mandatory BLE provisioning uses the 15.4 MAC if the chosen combo application is 15.4 MAC
Thread Stack Mandatory BLE provisioning uses the Thread stack if the chosen combo application is Thread.

Component Configuration

The BLE provisioning component facilitates user selection of a combo application configuration type. The combo application configuration type depends on the respective stacks, which are activated upon selection. For example, if the user selects the application configuration type as Thread, then the Thread stack will be activated.
If the user selects 'Thread' as the combo application configuration, they can choose 'FTD' and 'MTD' as the device roles. Also, for the 'MTD' selection, the user can enable the 'Sleepy End Device' option to allow sleep mode in the 'MTD'.

Generating Project

This section explains the steps required to develop an application based on the BLE Provisioning MCC component using MPLAB X Code Configurator.

Note: It is recommended that new users of MPLAB Code Configurator go through the overview

  • Create a new MCC Harmony Project. For more details, refer to 2.5 Creating a New MCC Harmony Project

  • Click Plus Symbol on the BLE PROVISIONING component in the Device Resources .The component is part of Wireless - System Services.

  • Upon BLE Provisioning Component being added to project graph, the different component dependencies will be requested to be added. The user has to select "yes" to add all the dependent components and for all attachment auto-connect requests. Right Click on TIME module for selecting the timer source, Select any of the timer. The project graph will look like the diagram below.

  • Click on the ‘BLE PROVISIONING‘, then use the configuration menu to set up the component according to user needs. Follow the steps outlined in the Component Configuration section.

  • For the WolfCrypt Library Component, enable "Support_ECB_Mode" and "Support Hardware Acceleration," and disable "Support Random Number Generation.

  • In Advertising Data menu, ensure that the Local Name is deselected and selected in Scan Response Data --> Local Name. In Advertising Data menu, ensure that Service Data is selected, the Service UUID is selected as 0xFEDA and the Service Data is set to 0xFF03 as shown in the figure. Also, set the device name and advertising interval according to the user's needs.

  • This step is applicable only if the device is enabled as a sleep end device. Select the clock configurations from the plug-ins and make the following changes to the clock configuration as shown in the pictures below. Also, set the PMU mode to BUCK_PWM mode to achieve lower power consumption; these settings are available as part of Device Support.

  • Add a console component to facilitate application console prints if required. Users can trigger the console by clicking on the CONSOLE component. The component is a part of System Services

    Right Click on instance0 UART for selecting the uart source, Select SERCOM0.

  • Click on SERCOM 0 => Go to configuration options => Change Receive pinout, Transmit pinout.
  • Configure the SERCOM0 system setting to enable the Direct High Speed inside system Configuration options → Generate Fuse settings → DEVCFG1.

  • Click on Generate Tab for Code generation. Upon code generation, Thread CoAP Component related source files will be added to the project. For more details on code generation, refer to Generate Code

  • Once Generation completes the header, source files of BLE provision service will be added under Header and Source Files based on the configuration.

  • Please perform the manual code configurations as mentioned below in the respective files:
    • Open the app.c file and include the header file app_prov.h #include "app_prov/app_prov.h". Also, insert the following code snippet after the APP_BleStackInit() call within the APP_Tasks function as in the below figure.
       APP_Prov_TRPS_Init();
      APP_Prov_TRPS_CommissionOnReset()
    • Open the app_ble_handler.c file located in app_ble project folder and include the header file by adding #include "../app_ble_conn_handler.h". Also, include the following code in the respective functions as shown in the figure.
       APP_BleGapConnEvtHandler(p_event);
         if (p_event->eventField.evtPairingComplete.status != BLE_SMP_PAIRING_SUCCESS)
            {
                BLE_GAP_Disconnect(p_event->eventField.evtPairingComplete.connHandle, GAP_DISC_REASON_REMOTE_TERMINATE);              
            }
    • Open the app_trsps_handler.c file located in app_ble project folder and include the header file by adding #include "#include "app_trps/app_trps.h". Add the following code in APP_TrspsEvtHandler() function as shown in figure. 
      APP_TRPS_EventHandler(p_event);

    • Open app_user_edits.c file. Comment out or remove the #error line. Update the freertos_hooks.c as mentioned in app_user_edits.c file. For more details , refer to 14.1  User Action.

  • Switch to MPLAB X IDE window and Build Project. The project should compile successfully. If user is getting following error “c:166: undefined reference to `g_adcTiming08” when adding the BLE Stack, please enable the "Remove unused sections" option in the project properties under xc32-ld options.
