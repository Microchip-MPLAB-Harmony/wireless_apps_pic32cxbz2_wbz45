Clone the Harmony repositories using MCC Content Manager Wizard

  1. 1.

    Create a new “MCC Harmony” project (In order to clone the Harmony repositories user needs to create an empty project and clone the required repositories)

    1. a.

      Select "New Project"

      Figure 1. New Project Creation
    2. b.
      Select "32-bit MCC Harmony Project"
      Figure 2. MCC Harmony Project option.
    3. c.
      Framework Selection
      Figure 3. Framework Selection.

      Local PC directory where cloned repos would be placed

    4. d.
      Project Settings
      Figure 4. Project Settings.

      Project Folder Path and Name Selection

    5. e.
      Configuration Settings and select the device WBZ451
      Figure 5. Configuration Settings.
      Device Name and Configuration Settings
  2. 2.
    Open “MPLAB Code Configurator” after creation of project. MCC Content Manager window will appear in IDE
    Figure 6. MCC in IDE.

  3. 3.
    Select “MPLAB Harmony” from the content manager window
    Figure 7. MCC Content Manager window.

    Select "Optional Content" as shown below
    Figure 8. Optional Content Window MCC.
  4. 4.
    Ensure all components are selected as displayed in the image below. Ignore version selection for now it will be taken care of at a later stage
    Figure 9. Required Content.
  5. 5.
    Select “Finish” The downloading of selected components from harmony repositories will take some minutes.If all the selected components are cloned successfully, MCC logs in IDE will display this information.
    Figure 10. MCC Logs Snapshot when all selected components are cloned successfully.
    Attention:
  6. If in the process a failure to download a particular component appears, try redownloading again only the component that failed to download. For example, if user received a prompt from MCC saying “dev_packs” was not download
  7. close the MCC reopen MCC again and start from step 3 in this section, with the “dev_packs” as the only missing component for downloading and select “Finish”
Figure 11. "dev_packs" being redownloaded.
  • 6.
    Select/Confirm the Harmony Framework Path
    Figure 12. Harmony Framework Path Selection.

    Note: Framework Path selection window appears beneath the Initializing Project pop-up, users should Choose Path as “C:\mchp”

    Figure 13. Framework Selection Path.
    Project Graph Window will appear after choosing the framework path and successful initialization of project
    Figure 14. Project Graph.
  • 7.
    Change to contentversions recommended for use during development in the table using the content manager from Device Resources and select “Apply”. This step is important and should be followed by user as every wireless stack and app example release is tested with certain version of dependant component.
    Figure 15. Changing content versions

    • After successfully checking out the right version of content/components, users can start developing application examples.
